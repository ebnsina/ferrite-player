Documentation

One <ferrite-player> tag for HLS, DASH, MP4, and live. An adaptive engine powers streaming and loads only when a manifest needs MSE.

Early release

Ferrite Player is 0.0.1 — core playback and the default skin are solid, but the API may still evolve. Pin a version in production.

Install

pnpm add ferrite-player

Then import it once — importing registers the <ferrite-player> custom element:

import 'ferrite-player';

Or straight from a CDN, no build step:

<script type="module" src="https://esm.sh/ferrite-player"></script>

Quickstart

<ferrite-player src="https://cdn.example.com/master.m3u8"></ferrite-player>

The default skin — play, scrubbable seek, volume, quality, captions, PiP, fullscreen, live badge, keyboard shortcuts — is included. Point src at any .m3u8, .mpd, or .mp4; the player picks native playback where it can and lazy-loads the adaptive engine for the rest.

Attributes

AttributeTypeDescription

JavaScript API

const player = document.querySelector('ferrite-player');

// Load with options — live, low-latency, DRM:
await player.load('https://cdn.example.com/manifest.mpd', {
  lowLatency: true,
  drm: { servers: { 'com.widevine.alpha': 'https://license…' } },
});

player.play();          // returns a Promise
player.pause();
player.src;             // current source (getter)
player.src = url;       // setter → loads
MethodDescription
load(src, opts?)Load a source with optional LoadOptions (live, lowLatency, drm, autoplay, subtitles, thumbnails, chapters).
addSubtitles(list)Add WebVTT/SRT caption tracks after load.
play() / pause()Control playback.
on(cb)Subscribe to state changes; returns an unsubscribe function.
stateA read-only snapshot of the current PlayerState.

State & events

Subscribe with on() and read player.state — a plain snapshot updated on every change:

const off = player.on(() => {
  const s = player.state;
  console.log(s.currentTime, s.duration, s.paused, s.live, s.qualities);
});
// later: off();

PlayerState includes: src, engine ('native'|'mse'), paused, waiting, currentTime, duration, buffered, volume, muted, rate, fullscreen, pip, live, atLiveEdge, qualities, currentQuality, textTracks, currentText, audioTracks, currentAudio, chapters, casting, and error.

Events

Beyond the state snapshot, the element fires discrete ferrite:* DOM events — the hook analytics integrations listen on. They bubble and cross the shadow boundary, so you can listen on the element or an ancestor.

player.addEventListener('ferrite:play', () => beacon('play'));
player.addEventListener('ferrite:seeked', (e) => beacon('seek', e.detail.currentTime));
player.addEventListener('ferrite:qualitychange', (e) => beacon('quality', e.detail.id));
player.addEventListener('ferrite:error', (e) => beacon('error', e.detail.message));
EventWhen
loadstart · loadedmetadataA source starts / finishes loading metadata
play · pause · playing · waiting · endedPlayback lifecycle
seeking · seeked · timeupdatePosition changes (detail.currentTime)
qualitychange · audiochange · texttrackchangeTrack selection (detail.id)
ratechange · volumechangeRate / volume changes
errorPlayback failed (detail.message)

Audio tracks

Multi-language / multi-track audio is detected automatically and appears as an Audio section in the settings menu — no configuration. Read state.audioTracks / state.currentAudio to drive your own UI.

Chapters

Pass a WebVTT chapters file — markers appear on the seek bar and the chapter title shows in the hover preview. Same format as any WebVTT, with each cue's text as the title.

await player.load(url, { chapters: '/chapters.vtt' });

Casting

AirPlay (Safari) and Chromecast (Chromium) buttons appear automatically when a receiver is available — the Cast SDK is lazy-loaded on demand, never in the core bundle. Picture-in-Picture works everywhere it's supported.

Theming

Style with CSS custom properties — no shadow-DOM piercing needed:

ferrite-player {
  --fp-accent: #ff3e00;   /* controls + progress */
  --fp-radius: 14px;      /* corner radius */
}

In React with TypeScript, add ferrite-player to your JSX intrinsic elements (or cast) for full typing.

Live streaming

Pass live/lowLatency at load. A live badge appears; when the viewer drifts back, it dims — one click jumps to the live edge.

await player.load('https://cdn.example.com/live.m3u8', { lowLatency: true });

DRM

Pass license servers (and a FairPlay cert for Safari) at load time — nothing is hardcoded in the bundle.

await player.load(url, {
  drm: {
    servers: {
      'com.widevine.alpha': 'https://license.example.com/widevine',
      'com.microsoft.playready': 'https://license.example.com/playready',
    },
    fairplay: { licenseUrl: 'https://…/fps', certificateUri: 'https://…/cert' },
  },
});

Subtitles (VTT & SRT)

Side-load captions in either format — SubRip is converted to WebVTT internally, so .srt and .vtt behave identically on both the native and adaptive engines. In-manifest tracks show up automatically; these add to them.

// at load time…
await player.load(url, {
  subtitles: [
    { url: '/caps/en.vtt', language: 'en', label: 'English', default: true },
    { url: '/caps/es.srt', language: 'es', label: 'Español' },
  ],
});

// …or any time after
await player.addSubtitles([{ url: '/caps/fr.srt', language: 'fr', srt: true }]);

Tracks appear in the settings menu and via the CC button; the last-used language is remembered across sessions.

Hover thumbnails

Pass a WebVTT storyboard and the seek bar shows a preview image as you hover or scrub — the format the Ferrite transcoder emits. Each cue's payload is an image URL with an optional #xywh sprite fragment.

await player.load(url, { thumbnails: '/storyboard.vtt' });

// storyboard.vtt
WEBVTT

00:00:00.000 --> 00:00:03.000
sprite.jpg#xywh=0,0,160,90

Without a storyboard you still get a time bubble on hover; with one you get frame previews.

Browser support

Modern evergreen browsers. HLS plays natively on Safari/iOS; DASH and non-native HLS use MSE + EME. MP4 plays natively everywhere. FairPlay on Safari, Widevine/PlayReady elsewhere.
